Scope & methodology

A combined active penetration test and passive vulnerability assessment was conducted across an investment firm's on-premises infrastructure, public-facing web properties, and cloud-hosted platforms. Active testing involved real exploitation attempts using industry-standard offensive tooling. Passive assessment covered seven distinct targets using automated scanning without direct exploitation.

Key findings — active pentest

Critical Domain Controller fully compromised — Zerologon CVE-2020-1472 CVSS 10.0

The SAGE$ machine account (Domain Controller) was vulnerable to Zerologon — a cryptographic flaw in the Netlogon protocol allowing an unauthenticated attacker to reset the DC machine account password to blank, then authenticate as the DC itself. Full domain dominance was achieved in a single exploit chain.

Windows Remote Management was accessible externally on port 5985 using unencrypted HTTP. Brute-force testing was conducted via CrackMapExec across a large credential list, confirming the attack surface for remote command execution without requiring physical access.

Impact
Remote command execution · Lateral movement · Credential interception (HTTP, unencrypted)
Remediation
Disable HTTP listener (5985) · Enable HTTPS (5986) · Restrict to admin jump hosts via firewall
High WordPress admin panel publicly accessible — no access controls CVSS 7.5

The /wp-admin/ and /wp-login.php pages returned HTTP 200 with no IP restriction, rate limiting, or MFA. The XML-RPC endpoint (/xmlrpc.php) was also active — enabling brute-force amplification via system.multicall and potential DDoS via pingback abuse.

Impact
OWASP A01:2021 — Broken Access Control · Credential stuffing · Plugin exploit path · CMS takeover
Remediation
IP whitelist /wp-admin/ · Enforce MFA · Disable xmlrpc.php · Enable login attempt limiting
High TLS certificate expired — HTTPS outage risk for public domain CVSS 7.5

SSL certificate verification confirmed an expiry of November 11, 2025. Post-expiry, HTTPS connections would fail across all browsers, making the investment platform completely inaccessible via secure connection and triggering client-side security warnings.

$ openssl s_client -connect [REDACTED-DOMAIN]:443 -servername [REDACTED-DOMAIN] Not Before: Dec 18, 2024 Not After : Nov 11, 2025 ← EXPIRED
Medium SMB signing disabled — NTLM relay attack path confirmed CVSS 5.3

SMB signing was not enforced on the domain controller (192.168.0.15). With SMB signing disabled, an attacker positioned on the network can intercept and relay NTLM authentication, escalating privileges without needing plaintext credentials — chaining directly with the Zerologon finding.

Impact
NTLM relay · SMB man-in-the-middle · Session hijacking · Privilege escalation without credentials
Remediation
Enforce SMB signing via Group Policy · Prefer Kerberos over NTLM · Restrict SMB to trusted hosts

Attack chain — initial access to full domain compromise

01

Reconnaissance & enumeration

Nmap + CrackMapExec used to enumerate live hosts, open ports (445/SMB, 5985/WinRM, 3389/RDP), OS fingerprint (Windows Server 2019), and domain membership. User accounts discovered via smbclient share enumeration.

02

Zerologon exploit — DC machine account reset

CVE-2020-1472 exploited against the SAGE$ Domain Controller. The Netlogon authentication flaw allowed the machine account password to be reset to blank with zero credentials required — no prior foothold needed.

03

Credential access — NTLM hash dump

Authenticated to the DC using the compromised SAGE$ account. secretsdump.py extracted the Administrator NTLM hash, confirming the blank password exploitation was successful and that all domain credentials are accessible.

04

Pass-the-hash — SYSTEM shell obtained

psexec.py used to authenticate with the dumped NTLM hash, obtaining an interactive SYSTEM-level shell on the Domain Controller. Full domain dominance confirmed — all AD objects, GPOs, and connected resources under attacker control.

05

Findings documented — remediation handed to client

All exploitation evidence captured with live terminal output. CVSS-scored report with immediate, short-term, and long-term remediation roadmap delivered in executive and technical formats.

Remediation roadmap

Immediate (0–24 hours)

  • Isolate the Domain Controller from the network and apply patch KB4570334 to close the Zerologon vulnerability
  • Emergency reset of all privileged domain accounts (Domain Admins, Enterprise Admins, service accounts)
  • Disable WinRM HTTP listener (port 5985); restrict remote management to TLS-encrypted channels only

Short-term (1–7 days)

  • Rebuild or restore the Domain Controller from a known-clean backup — assume full compromise of the existing instance
  • Enable SMB signing via Group Policy across all domain-joined systems to prevent NTLM relay attacks
  • Implement Local Administrator Password Solution (LAPS) to randomise and rotate local admin credentials
  • Restrict /wp-admin/ and /wp-login.php via IP whitelist; enforce MFA; disable xmlrpc.php
  • Renew TLS certificate and enable auto-renewal; configure HSTS across all web properties

Long-term security measures

  • Deploy Privileged Access Workstations (PAWs) for all administrative tasks to reduce lateral movement risk
  • Implement Microsoft Defender for Identity to detect Zerologon-style attacks and anomalous DC authentication
  • Publish DMARC (p=reject), DKIM, and validate SPF across all sending domains to prevent email spoofing
  • Conduct red team exercises simulating machine account attacks and NTLM relay chains annually
